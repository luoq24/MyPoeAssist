## 一、产品概述

### 1.1 产品定位

MyPoeAssist 是一款面向《流放之路 2》（POE2）**国服玩家**的交易辅助桌面工具。核心解决两个问题：

1. **"好价难遇"** — 市场上低价商品转瞬即逝，玩家不可能一直盯盘
2. **"词缀看不懂"** — 石板、技能宝石等道具词缀繁多，缺乏直观的价值参考

### 1.2 目标用户

- POE2 国服（腾讯服）玩家
- 需要频繁在交易站买卖道具的中高级玩家
- 希望用最低价格购入特定道具的"捡漏"玩家

### 1.3 核心价值

| 场景 | 痛点 | MyPoeAssist 的解法 |
|------|------|-------------------|
| 盯盘买装备 | 需要反复刷新交易站，手动比价 | 自动监控，低价出现播放音效提醒，通知面板集中处理 |
| 遇到好价但不在藏身处 | 切角色太慢，卖家已被抢 | 通知面板中点击"去买"，一键复制传送链接，回到藏身处后秒传 |
| 看不懂石板词缀价值 | 不知道哪些词缀值钱，容易卖亏 | 指定道具类型，自动统计所有词缀的市场价格分布 |
| 刷地图时看不懂引路石词缀好坏 | 词缀数量多，前后缀混合难以快速判断 | **✅ 已实现** — 剪贴板监听 + 词缀解析 + 音效提醒，详见 [模块四：引路石词缀分析与监控](./引路石词缀分析与监控.md) |

## 二、功能模块详细设计

### 模块一：市场监控与价格提醒

#### 2.1 功能描述

用户预先设定需要监控的商品条件（物品类型、词缀要求、价格上限等），工具在后台持续轮询 POE2 国服交易站 API。当出现满足条件的低价商品时，播放自定义提醒音效，并将商品信息写入应用内的通知面板，供用户集中处理。

#### 2.2 用户操作流程

```
用户创建监控任务
    ↓
选择监控方式（二选一）
    ├─ 方式A：手动配置 — 选择物品类型 + 词缀 + 价格上限
    └─ 方式B：导入链接 — 粘贴已设置好筛选的交易站网页URL
    ↓
工具解析并确认监控条件
    ↓
开始监控（后台轮询）
    ↓
发现符合条件的商品 → 播放提醒音效 + 写入通知列表
    ↓
用户切回应用查看通知面板
    ├─ [去买] 一键前往 → 利用网页链接传送至卖家藏身处
    ├─ [忽略] 标记为已读，保留在列表中
    └─ [删除] 从列表中移除
```

#### 2.3 监控任务配置

**方式A：手动配置**

| 配置项 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| 任务名称 | 否 | 自定义名称，便于管理 | "低价克己权杖" |
| 物品类型 | 是 | 从下拉列表选择物品大类/子类 | "权杖" → "克己权杖" |
| 物品名称 | 否 | 指定暗金/传奇物品名称 | "森林灵像" |
| 词缀条件 | 否 | 指定必须包含的词缀及数值范围 | "+30% 以上火焰抗性" |
| 价格上限 | 是 | 超过此价格的商品不提醒 | "50 混沌石" |
| 货币类型 | 是 | 价格参照的通货类型 | 混沌石 / 崇高石 |
| 仅在线卖家 | 否 | 是否只关注在线玩家的挂牌 | 默认开启 |

**方式B：导入交易站链接**

用户在 POE2 国服交易站（`https://poe.game.qq.com/trade2`）上设置好筛选条件后，复制浏览器地址栏的完整 URL，粘贴到工具中。工具自动解析 URL 中的查询参数，提取出搜索条件。

| 解析内容 | 说明 |
|----------|------|
| 物品类型 / 名称 | 从 URL 路径或 query 参数提取 |
| 筛选器条件 | 解析 URL 中携带的 filter 参数 |
| 联赛 | 自动匹配当前赛季 |
| 排序方式 | 按价格升序 |

> **URL 格式参考**：`https://poe.game.qq.com/trade2/search/poe2/奥杜尔秘符?q=...`

#### 2.4 监控策略

| 参数 | 值 | 说明 |
|------|-----|------|
| 轮询间隔 | 30~60 秒（可配置） | 需遵守 API 频率限制 |
| 结果去重 | 基于商品 ID 去重 | 同一商品仅提醒一次 |
| 结果缓存 | 缓存最近 24h 已提醒的商品 ID | 防止重启后重复提醒 |
| 最大并行任务 | 5 个（初始） | 受限于 API 频率限制 |
| 价格阈值 | 严格低于用户设定价格 | 仅低价商品触发提醒 |

#### 2.5 提醒机制

**通知策略：音效提醒 + 通知面板**

当监控引擎发现符合条件的低价商品时，不弹出桌面弹窗，而是：
1. **播放提醒音效** — 用户可自定义音效文件（支持 `.wav` / `.mp3`），播放一次
2. **写入通知面板** — 新通知出现在应用内的"通知"子面板中，以未读状态高亮显示

> 这种设计避免了弹窗打断游戏操作，玩家可以在死亡/城镇/藏身处等间隙切回应用集中处理。

**通知面板：**

```
┌─────────────────────────────────────────────────────────────┐
│ 📋 通知列表                               [全部已读] [清空] │
├─────────────────────────────────────────────────────────────┤
│ ● 克己权杖 - 森林灵像    35C（目标50C）  卖家:PlayerA  2分钟前 │
│   [去买]  [忽略]  [删除]                                    │
├─────────────────────────────────────────────────────────────┤
│ ● 精华 - 愤怒精华       12C（目标20C）  卖家:PlayerB  5分钟前 │
│   [去买]  [忽略]  [删除]                                    │
├─────────────────────────────────────────────────────────────┤
│   项链 - 碧影护身符      80C（目标100C） 卖家:PlayerC  12分钟前│
│   [去买]  [忽略]  [删除]                                    │
└─────────────────────────────────────────────────────────────┘

● = 未读通知（高亮背景）    空心圆 = 已读通知（普通背景）
```

**通知条目数据结构：**

| 字段 | 说明 |
|------|------|
| 物品名称 | 道具全名（类型 + 具体名称） |
| 价格 | 挂牌价格及货币类型 |
| 目标价格 | 用户设定的价格上限 |
| 卖家名称 | 卖家角色名 |
| 发现时间 | 通知产生的时间 |
| 读取状态 | 未读 / 已读 |
| 关联任务 | 产生该通知的监控任务名称 |

**通知条目操作：**

| 操作 | 说明 |
|------|------|
| **去买** | 将交易站该商品的页面链接复制到剪贴板，用户可在游戏内浏览器中打开并传送至卖家藏身处 |
| **忽略** | 标记为已读，通知保留在列表中但不再高亮 |
| **删除** | 从通知列表中永久移除 |
| **（自动）过期** | 同一卖家的同一商品再次出现时，旧条目自动标记为过期（见下方规则） |

**同质化消息处理规则：**

当监控引擎产生新通知时，检查通知列表中是否已存在同质化条目，若存在则自动标记旧条目为"过期"。

> 判定条件：同一监控任务下，**卖家名称相同** 且 **商品 ID 相同** 的两条通知视为同质化消息。

```
新通知到达
    ↓
在通知列表中查找：同任务 + 同卖家 + 同商品ID
    ↓
┌─ 找到旧条目 ─→ 将旧条目标记为"过期"（灰色 + 删除线样式）
│               → 旧条目的操作按钮变为 [已过期]（不可交互）
│               → 仅保留删除操作
└─ 未找到旧条目 → 正常插入新通知
```

**通知条目状态汇总：**

| 状态 | 样式 | 说明 |
|------|------|------|
| 未读 | 高亮背景 + 粗体 | 新到通知，等待用户处理 |
| 已读 | 普通背景 | 用户点击了"忽略" |
| 过期 | 灰色背景 + 删除线 | 同一卖家同一商品的新通知已到达，此条目自动失效 |

> **设计意图**：卖家对同一商品可能多次调价、重新上架。当新通知到来时，旧通知的价格信息已失去参考意义，自动过期可避免用户误操作（如点击过时的"去买"导致商品已售）。

**音效设置：**

| 配置项 | 说明 |
|--------|------|
| 音效文件路径 | 用户自定义音效文件（`.wav` / `.mp3`），由用户后续上传 |
| 音量 | 0%~100%，默认 80% |
| 重复播放 | 可选：单次 / 每隔 N 秒重复直到用户查看 |
| 静音时段 | 可设置免打扰时间段 |

#### 2.6 任务管理

| 操作 | 说明 |
|------|------|
| 新建任务 | 创建新的监控任务 |
| 暂停/恢复 | 临时暂停某个任务的监控 |
| 编辑任务 | 修改监控条件（物品、词缀、价格等） |
| 删除任务 | 永久删除监控任务 |
| 任务日志 | 查看每个任务的历史触发记录（最近 100 条） |
| 导入/导出 | 将监控任务配置导出为文件，方便分享或备份 |

### 模块二：词缀价值统计

#### 2.7 功能描述

用户指定某个道具类型（如"石板"），工具自动从交易站采集该类型下所有挂牌商品的词缀和价格数据，经过统计分析后，展示每个词缀的市场价值分布，帮助玩家判断道具的价值。

#### 2.8 用户操作流程

```
选择道具类型（如"石板"）
    ↓
设置统计参数
    ├─ 统计范围：最近 24h / 7d / 全量
    ├─ 最低样本量：过滤掉样本不足的词缀
    └─ 货币计价：混沌石 / 崇高石
    ↓
开始采集（后台批量拉取交易站数据）
    ↓
数据统计与聚合
    ↓
展示统计结果
    ├─ 词缀价值排行榜
    ├─ 单个词缀的价格分布
    └─ 组合词缀价值参考
```

#### 2.9 统计维度

**词缀价值排行**

| 字段 | 说明 |
|------|------|
| 词缀名称 | 如 "+X% 火焰抗性" |
| 样本数量 | 该词缀出现在多少件挂牌商品中 |
| 中位价格 | 该词缀商品的价格中位数 |
| 均价 | 该词缀商品的平均价格 |
| 价格区间 | P25 ~ P75 的价格范围 |
| 价值评级 | 综合评分：S / A / B / C / D |

**价格分布图**

对选定词缀展示价格分布直方图：
- X 轴：价格区间
- Y 轴：商品数量
- 辅助线：标注中位数、均价

**词缀组合分析**

分析哪些词缀组合出现时，道具价值显著提升：
- 双词缀组合 TOP 10
- 三词缀组合 TOP 10
- 组合溢价率（组合价格 vs 单词缀价格之和）

#### 2.10 支持的道具类型（首批）

| 道具类型 | 说明 | 优先级 |
|----------|------|--------|
| 石板（Tablets） | POE2 地图强化道具，词缀价值差异大 | P0 |
| 技能宝石 | 不同等级和品质的价值差异 | P1 |
| 精华（Essences） | 固定词缀的通货类道具 | P1 |
| 戒指 / 项链 | 首饰类装备词缀丰富 | P2 |
| 武器 | 武器类词缀价值参考 | P2 |
| 护甲 | 防具类词缀价值参考 | P2 |

#### 2.11 数据采集策略

| 策略 | 说明 |
|------|------|
| 采集方式 | 调用国服交易站 API，按价格升序批量拉取 |
| 每次拉取量 | 每批 10 条（API 限制），循环拉取直到样本量足够 |
| 样本量目标 | 每个道具类型至少 200 件商品 |
| 频率限制 | 遵守 API 限流策略，预计完整采集耗时 5~15 分钟 |
| 数据缓存 | 采集结果本地缓存，避免重复请求 |
| 刷新策略 | 支持手动刷新 + 定时自动刷新（默认每 4 小时） |

### 模块三：用户认证与国服适配

#### 2.13 QQ/微信登录

由于 POE2 国服（腾讯服）API 要求 Cookie 认证，工具需要实现完整的腾讯 OAuth 登录流程。

**登录流程：**

```
首次启动 / Cookie 过期
    ↓
弹出登录窗口（PyQt6 QWebEngineView）
    ↓
导航到 https://poe.game.qq.com/trade2（自动跳转腾讯登录页）
    ↓
用户选择 QQ 或微信扫码登录
    ↓
登录成功 → 从 QWebEngineView CookieStore 捕获完整 Cookie
    ↓
加密存储到本地数据库
    ↓
后续请求自动携带 Cookie
    ↓
POETOKEN 过期（约 1 天）→ 自动刷新或提示重新登录
```

**关键 Cookie 管理：**

| Cookie | 有效期 | 管理策略 |
|--------|--------|---------|
| `POESESSID` | 随会话 | 登录后自动获取 |
| `POETOKEN`（JWT） | 约 1 天 | 监控 `Set-Cookie` 响应头自动刷新 |
| `RK` / `ptcz` / `eas_sid` | 腾讯系长期态 | 随登录一并获取 |

#### 2.14 联赛管理

| 功能 | 说明 |
|------|------|
| 自动获取联赛列表 | 从国服 API 或页面配置获取当前可用联赛 |
| 联赛选择 | 用户可手动选择要监控的联赛（奥杜尔秘符 / 永久 等） |
| 赛季切换 | 新赛季开始时自动提示更新 |

### 模块四：引路石词缀分析与监控（已实现）

> 详见：[引路石词缀分析与监控](./引路石词缀分析与监控.md)

本模块是 MyPoeAssist **目前已实现的核心功能**，通过监听系统剪贴板自动捕获游戏内复制的引路石信息，解析前后缀词缀数量并播放对应音效提醒，帮助玩家快速判断地图质量。包含三个子功能：

1. **引路石词缀实时分析** — 剪贴板监听 → 词缀解析 → 前后缀计数 → 音效提醒
2. **未知词缀采集收集** — 扫描新词缀追加到本地词缀库，扩展已知词缀覆盖范围
3. **透明覆盖层绘制** — 在游戏窗口上层绘制矩形标记（预留辅助功能）

### 3.2 国服 API 交互

基于 [腾讯服 API 知识库](../ref_docs/base_knownage.md) 中记录的端点：

| 操作 | API | 方法 |
|------|-----|------|
| 搜索商品 | `POST /api/trade2/search/poe2/{league}` | 构造 Query JSON，返回 query_id + 结果 ID 列表 |
| 获取详情 | `GET /api/trade2/fetch/{ids}?query={query_id}` | 批量获取商品挂牌详情 |
| 频率限制 | 遵循 `X-Rate-Limit-*` 响应头 | 动态限流，令牌桶算法 |

**关键注意事项：**

- 所有请求必须携带完整 Cookie 链
- 联赛名需 URL 编码（如 `%E5%A5%A5%E6%9D%9C%E5%B0%94%E7%A7%98%E7%AC%A6`）
- 错误码通过 JSON 响应体 `error.code` 返回（非 HTTP 状态码）
- 频繁无效请求会触发临时封禁

### 3.3 数据存储

| 数据 | 存储方式 | 说明 |
|------|----------|------|
| 监控任务配置 | SQLite | 任务列表、筛选条件、价格阈值 |
| 已提醒商品 ID | SQLite | 24h 过期的去重记录 |
| Cookie 加密数据 | SQLite + AES | 登录凭证持久化 |
| 词缀统计数据 | SQLite + 本地缓存 | 采集结果缓存，支持定时刷新 |
| 用户设置 | JSON 配置文件 | 联赛选择、通知偏好、轮询间隔等 |

### 3.4 API 频率限制适配

| 策略 | 说明 |
|------|------|
| 令牌桶限流 | 解析响应头中的 `X-Rate-Limit-*` 字段，动态调整请求速率 |
| 请求间隔 | 基础间隔 300ms，根据限流状态自动调整 |
| 429 处理 | 指数退避，等待 `Retry-After` 秒数后重试 |
| 批量请求合并 | 多个监控任务共享同一个限流器，避免触发封禁 |
| 错误请求抑制 | 连续错误超过阈值时自动暂停，避免触发无效请求封禁 |

## 四、非功能性需求

### 4.1 性能要求

| 指标 | 目标 |
|------|------|
| 监控延迟 | 从商品上架到通知弹出 ≤ 90 秒（含轮询间隔） |
| 启动时间 | 冷启动 ≤ 5 秒 |
| 内存占用 | 常驻内存 ≤ 200MB |
| CPU 占用 | 后台监控时 CPU 占用 ≤ 2% |

### 4.2 可靠性

| 需求 | 说明 |
|------|------|
| 网络异常恢复 | API 请求失败时自动重试（最多 3 次），不丢失监控任务 |
| Cookie 过期处理 | POETOKEN 过期后自动尝试刷新，失败时弹窗提示重新登录 |
| 数据持久化 | 所有用户配置和监控任务在应用重启后恢复 |
| 崩溃恢复 | 应用异常退出后，下次启动自动恢复之前的监控状态 |

### 4.3 安全性

| 需求 | 说明 |
|------|------|
| Cookie 加密存储 | 使用 AES 加密存储到本地 SQLite，不明文保存 |
| 无密码存储 | 不存储用户 QQ/微信密码，仅保存登录后的 Cookie |
| 隐私保护 | 不上传任何用户数据到第三方服务器 |

### 4.4 兼容性

| 需求 | 说明 |
|------|------|
| 操作系统 | Windows 10/11（64 位） |
| Python 版本 | Python 3.10+ |
| UI 框架 | PyQt6（含 QWebEngineView，用于 OAuth 登录） |
| 音效播放 | PyQt6 QMediaPlayer（系统原生解码，支持 .wav / .mp3） |
