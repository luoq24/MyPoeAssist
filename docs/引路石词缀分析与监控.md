# 模块四：引路石词缀分析与监控

> 状态：**已实现**  
> 版本：v0.1  
> 源码位置：`source/models/model.py`、`source/models/mod_collector.py`、`source/views/windows/item_helper.py`、`source/views/windows/mod_collect.py`、`source/tools/soundPlayer.py`

---

## 4.1 功能概述

本模块是 MyPoeAssist **目前已实现的核心功能**，面向 POE2 国服玩家在游戏内快速判断引路石（地图）词缀质量。通过监听系统剪贴板，自动捕获玩家在游戏内复制的引路石信息，解析其前后缀词缀数量，并根据词缀组合播放不同的音效提醒，帮助玩家快速决策地图是"继续刷"还是"丢弃"。

---

## 4.2 功能组成

本模块包含三个子功能：

| 子功能 | 说明 | 状态 |
|--------|------|------|
| 引路石词缀实时分析 | 监听剪贴板，自动解析引路石词缀数量并音效提醒 | ✅ 已实现 |
| 未知词缀采集收集 | 扫描新词缀并追加到本地词缀库 | ✅ 已实现 |
| 透明覆盖层绘制 | 在游戏窗口上绘制矩形标记（辅助功能） | ✅ 已实现 |

## 4.3 引路石词缀实时分析

### 4.3.1 工作原理

```
玩家在游戏内 Ctrl+C 复制引路石
    ↓
工具检测到剪贴板变化
    ↓
验证当前前台窗口是否为 POE2（POEWindowClass）
    ↓
验证剪贴板内容是否以 "物品类别: 引路石" 开头
    ↓
解析词缀区域（分隔符 '--------\n' 分割后的第 4 段）
    ↓
将每条词缀中的数字替换为 '#' 做标准化
    ↓
对照词缀库计算：前缀数、后缀数、深渊词缀数、坏词缀数、未知词缀数
    ↓
根据词缀数量组合，播放对应音效
    ↓
将分析结果（前缀数 + 后缀数）显示到界面
```

### 4.3.2 支持的地图稀有度

| 稀有度 | 说明 |
|--------|------|
| 魔法（Magic） | 蓝色地图，最多 1 前缀 + 1 后缀 |
| 稀有（Rare） | 黄色地图，最多 3 前缀 + 3 后缀 |

### 4.3.3 词缀分类与权重

词缀配置存储在 `data/mod/map.json` 中，分为以下五个类别：

| 类别 | 键名 | 说明 | 示例 |
|------|------|------|------|
| 前缀 | `prefix` | 地图中的正面/怪物增强词缀 | "怪物具有闪避"、"区域内有 # 个额外的飞升怪物群" |
| 后缀 | `subfix` | 地图中的负面/玩家削弱词缀 | "怪物暴击率提高 #"、"玩家的抗性上限 -#%" |
| 深渊词缀 | `shenyuan` | 深渊相关词缀（与前后缀不冲突，独立计数） | "区域被深渊占据 (desecrated)"、"深渊通向深渊首领 (desecrated)" |
| 坏词缀 | `bad` | 特别不受欢迎的词缀 | "玩家周期性受到时空锁链诅咒" |
| 新发现词缀 | `newfix` | 运行时发现的、尚未归类的新词缀 | 自动追加 |

每条词缀配置的值为权重（0 或 1），值为 0 表示该词缀当前不计入数量统计（如 "+#% 怪物暴击伤害加成"、"怪物攻击速度加快 #%"）。

### 4.3.4 音效提醒规则

根据前缀数、后缀数、深渊词缀数、坏词缀数的组合，播放不同的音效：

| 音效枚举 | 文件名 | 触发条件 | 含义 |
|----------|--------|----------|------|
| `Unknown` | 未知词缀.wav | 存在未知词缀 | 需要人工确认词缀分类 |
| `Bad` | 讨厌词缀图.wav | 存在坏词缀 | 地图有特别讨厌的词缀 |
| `Shenyuan` | 深渊地图.wav | 存在深渊词缀 | 深渊相关地图 |
| `Normal` | 普通地图.wav | 总词缀数 = 0 | 白图，无词缀 |
| `Magic` | 魔法地图.wav | 总词缀数 ≤ 2 | 低词缀量 |
| `Wait` | 待培养地图.wav | 不满足其他特殊条件 | 中等词缀，待培养 |
| `Full` | 满词缀地图.wav | 总词缀数 ≥ 6 | 词缀已满 |
| `Bad3` | 3前缀坏图.wav | 前缀数 ≥ 3 | 前缀过多，不好 |
| `Good3` | 3词好图.wav | 后缀 = 3 且前缀 = 0 | 3 词好图 |
| `Good4` | 4词好图.wav | 后缀 = 3 且前缀 = 1 | 4 词好图 |
| `Good5` | 5词好图.wav | 后缀 = 3 且前缀 = 2 | 5 词好图 |

**优先级逻辑**：未知 > 坏词缀 > 深渊 > 普通 > 魔法 > 满词缀 > 3前缀坏 > 好图

## 4.4 未知词缀采集收集

### 4.4.1 功能说明

用于扩展本地词缀库。开启"收集词缀"模式后，工具不再播放音效，而是将复制到的引路石词缀逐条与本地词缀库比对，将未收录的新词缀自动追加到 `newfix` 分类中，并实时显示在"收集Mod"标签页。

### 4.4.2 工作流程

```
开启"收集词缀"模式
    ↓
玩家 Ctrl+C 复制引路石
    ↓
解析词缀列表，逐条标准化（数字→#）
    ↓
与 prefix / subfix / newfix 字典比对
    ↓
若词缀不在任何分类中 → 追加到 newfix，实时显示
    ↓
玩家点击"save conf" → 将 newfix 持久化到 map.json
    ↓
玩家手动将 newfix 中的词缀归类到 prefix 或 subfix
    ↓
玩家点击"load conf" → 重新加载词缀库
```

### 4.4.3 用户界面（收集Mod标签页）

```
┌──────────────────────────────────────────┐
│  [load conf]  [save conf]  [print new]   │
├──────────────────────────────────────────┤
│                                          │
│  prefix-1 怪物具有闪避                     │
│  prefix-1 怪物伤害提高 #%                  │
│  ...                                     │
│  ----------------                         │
│  subfix-1 怪物暴击率提高 #%                │
│  ...                                     │
│  ----------------                         │
│  shenyuan-1 区域被深渊占据 (desecrated)    │
│  ...                                     │
│  ----------------                         │
│  bad-1 玩家周期性受到时空锁链诅咒           │
│  ----------------                         │
│  newfix-1 新发现的未知词缀                  │
│                                          │
└──────────────────────────────────────────┘
```

**操作按钮：**

| 按钮 | 功能 |
|------|------|
| load conf | 重新从 `map.json` 加载词缀配置 |
| save conf | 将当前 newfix 追加保存到 `map.json` |
| print new | 在控制台输出所有新发现的词缀列表 |

## 4.5 透明覆盖层（辅助功能）

### 4.5.1 功能说明

提供在 POE2 游戏窗口上层绘制半透明矩形框的能力，可用于标记地图物品位置等视觉辅助。通过 Windows API 创建一个透明的置顶窗口，使用 `WS_EX_LAYERED | WS_EX_TRANSPARENT` 样式确保不拦截游戏的鼠标/键盘输入。

### 4.5.2 核心能力

| 能力 | 说明 |
|------|------|
| 窗口定位 | 自动查找 POE2 窗口并创建等大覆盖层 |
| 矩形绘制 | 在指定坐标绘制红色空心矩形 |
| 清除标记 | 清空所有已绘制矩形 |
| 鼠标穿透 | 覆盖层不阻挡鼠标事件 |

> **当前状态**：核心绘制逻辑已实现，但尚未与主功能流程集成（`move_map_by_type` 方法中已预留调用入口但暂未激活）。

## 4.6 词缀标准化处理

所有词缀在比对前均经过标准化处理：将词缀文本中的所有连续数字替换为 `#` 占位符。

**示例：**

| 原始词缀 | 标准化结果 |
|----------|-----------|
| 怪物伤害提高 30% | 怪物伤害提高 #% |
| 区域内有 2 个额外的飞升怪物群 | 区域内有 # 个额外的飞升怪物群 |
| +15% 怪物的元素抗性 | +#% 怪物的元素抗性 |

这确保了同一词缀在不同数值下能匹配到同一条规则。
